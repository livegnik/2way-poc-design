



# 04 Contact and profile flow

## 1. Purpose and scope

Defines how contact and profile data is created and updated for identities, including authorization and visibility constraints.

## 2. Responsibilities and boundaries

This specification is responsible for:

* Defining the allowed mutations for contact/profile data.
* Declaring ownership and ACL requirements.

This specification does not cover:

* The specific schema fields for any app.
* UI presentation or contact discovery UX.
