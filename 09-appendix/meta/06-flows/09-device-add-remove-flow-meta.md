



# 09 Device add/remove flow

## 1. Purpose and scope

Defines how devices are enrolled and removed for an identity, including authorization and persistence requirements.

This specification references:

* [06-flows/09-device-add-remove-flow.md](../../../06-flows/09-device-add-remove-flow.md)

## 2. Responsibilities and boundaries

This specification is responsible for:

* Defining the device enrollment and removal steps.
* Declaring authorization and validation requirements.

This specification does not cover:

* Device UX or pairing UI.
* Platform-specific secure storage implementations.
