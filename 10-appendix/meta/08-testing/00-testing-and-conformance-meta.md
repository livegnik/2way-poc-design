## 2. Responsibilities and boundaries

This specification is responsible for:

* Declaring required test categories and conformance expectations.
* Declaring canonical suite matrix inputs for generated test-conventions references.
* Defining fuzz/soak bounds and completion criteria.
* Declaring canonical bounds matrix inputs for generated test-conventions references.
* Defining canonical performance sanity workload defaults for Build Plan perf tripwire tests.
* Defining authoritative CI suite command mappings for generated CI guidance.
* Defining the regression test policy.

This specification does not cover:

* Test implementation details or code.
